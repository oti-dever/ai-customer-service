## 四、选型说明
### 4.1 CMake（而不用 qmake）
+ Qt 6 推荐 CMake，qmake 处于维护阶段
+ 生态通用，文档、示例、第三方库多基于 CMake
+ 便于后续跨平台（如 Linux/macOS）

### 4.2 MSVC（而不用 MinGW）
+ Qt 官方提供 MSVC 预编译版，配置简单
+ 与 Windows 兼容性更好，调试体验成熟
+ 与 Qt Creator 配合时，在 Kits 中选 MSVC 即可

### 4.3 SQLite（而不用 MySQL）
+ 本地单机场景，数据量有限，无需独立数据库服务
+ 零部署：单文件，无安装、无端口占用
+ Qt 自带 SQL 模块，可直接使用

补充：现阶段项目为本地桌面应用，用户数量少，不需要用 MySQL，关键不是库的选型，而是“怎么存”。

### 4.4 JSON
+ AI API（OpenAI 兼容格式）请求/响应均为 JSON
+ 配置、扩展字段存储用 JSON，易读易维护
+ Qt 6 提供 QJsonDocument，无需第三方库

### 4.5 Qt Network（而不用 Asio 等第三方库）
+ 需求以 HTTP 调用 AI API 为主，Qt Network 足够
+ 与 Qt 事件循环、信号槽风格一致
+ 减少第三方依赖，随 Qt 升级维护

### 4.6 项目结构（分层架构）
+ 分层设计，UI、业务、数据解耦，便于维护和扩展
+ `services/` 可替换 AI 提供商或平台对接方式
+ `data/` 专注持久化，业务逻辑在 `core/`
