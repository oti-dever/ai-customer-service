## C++ + Python RPA 打包与启动策略

本说明文档用于约定当前阶段桌面端（Qt C++）与 Python RPA 组件的打包方式和启动流程，目标是让客服同事 **解压即可用 / 安装即可用**，无需手工配置复杂环境。

---

## 1. 目录结构建议

打包后的应用根目录建议类似：

```
app_root/
  demo1.exe                 # Qt 主程序
  database/
    app.db                  # SQLite 数据库（C++ 和 Python 共享）
  logs/
    yyyy-MM-dd.log          # Qt 端日志
  python/
    rpa/
      main.py               # Python Reader（示例）
      writer.py             # Python Writer（示例）
      ...                   # 将来可以有更多模块
  python-rpa.exe            # PyInstaller 打好的独立 RPA 进程（可选）
  QtPlugins/                # Qt 依赖
  ...
```

- **数据库文件** 放在项目根目录的 `database/` 子文件夹下（`database/app.db`），由 Qt 端自动创建；
- **日志文件** 放在项目根目录的 `logs/` 子文件夹下；
- Python 端通过相对路径（`Path(__file__).resolve().parents[2] / "data" / "app.db"`）或配置文件拿到同一 SQLite 文件路径。

---

## 2. Python 组件打包方式

为避免要求客服机器上单独安装 Python 解释器，建议使用如下方式之一：

- **PyInstaller**：
  - 将 `rpa/main.py` 与其依赖（OCR 库、自动化库等）打包成一个 `python-rpa.exe`；
  - 优点：部署简单，单个 exe 即可运行。
- **其他方案**（如 `cx_Freeze`）也可，但优先统一选用一种。

建议初期先保留源码形式（`rpa/*.py`），方便开发调试；到准备发给同事试用时再输出打包版本。

---

## 3. 启动策略

### 3.1 简单方案：手动启动 RPA

适合早期内测阶段：

1. 客服双击 `demo1.exe` 启动桌面程序；
2. 再双击 `python-rpa.exe`（或运行一个批处理/快捷方式），启动 Python RPA 进程；
3. 关闭顺序任意，程序退出后数据库内容仍保留。

优点：实现成本最低，易于观察 RPA 控制台输出（日志）。

### 3.2 进阶方案：Qt 程序自动拉起 RPA

后续可以在 Qt 端增加一个简单的启动逻辑（示意）：

- 在 `ConversationManager::initialize()` 或平台连接入口处：
  - 检查 RPA 进程是否已经存在（例如通过互斥量或简单的 `QProcess` 列表）；
  - 如未启动，则通过 `QProcess` 启动 `python-rpa.exe`；
  - 仅在 Windows 平台下启用该逻辑。

伪代码示例：

```cpp
QProcess* rpaProcess = new QProcess(this);
QString exePath = QCoreApplication::applicationDirPath() + "/python-rpa.exe";
rpaProcess->setProgram(exePath);
rpaProcess->start();
```

此方案下，客服只需启动一次 Qt 程序，RPA 会自动跟随运行。

## 4. 配置与路径约定

- **数据库路径**：
  - Qt 端：`QCoreApplication::applicationDirPath() + "/database/app.db"`（项目根目录的 `database/` 子文件夹）；
  - Python 端：`Path(__file__).resolve().parents[2] / "database" / "app.db"`（项目根目录的 `database/` 子文件夹）；
  - 也可通过环境变量 `RPA_DB_PATH` 或 `config.ini` 覆盖。
- **平台开关**：
  - 可在同一配置文件中为不同平台（`qianniu`, `pdd_web` 等）设置启用/禁用开关；
  - Python 端根据配置决定实际启动哪些 Reader/Writer 循环。

---

## 5. 日志与诊断

- Qt 端日志输出到项目根目录的 `logs/yyyy-MM-dd.log`；
- 建议 Python 端也将日志写到同一 `logs/` 目录（按进程名或模块名区分文件），方便排查：
  - 例如：`logs/python-rpa-reader.log`, `logs/python-rpa-writer.log`。
- 关键场景（如 SQLite 打不开、RPA 找不到窗口、发送失败）需明确记录。

---

## 6. 迭代方向

- 初期版本可以只依赖 **Reader + Writer 示例脚本**，用假消息和状态更新打通整条链路；
- 等千牛 / 拼多多 UI 自动化逻辑稳定后，再：
  - 用 PyInstaller 打出正式版 `python-rpa.exe`；
  - 在 Qt 中接好自动启动逻辑；
  - 补充配置界面，允许客服在 UI 中切换“是否启用 RPA”、“启用哪些平台”等选项。
