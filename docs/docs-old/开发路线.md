### Phase 1：核心消息管线（~1 周）

这是最关键的一步——把"消息流进来 → 显示在界面"的**骨干通路打通**。

| 任务           | 具体内容                                                     | 产出                       |
| -------------- | ------------------------------------------------------------ | -------------------------- |
| **数据层**     | `conversations` / `messages` 表 + `ConversationDao` + `MessageDao` | 消息持久化能力             |
| **接口层**     | `IPlatformAdapter` 抽象接口（connect/startListening/sendMessage/信号） | 平台适配器骨架             |
| **路由层**     | `MessageRouter`（接收适配器信号 → 写库 → 通知 UI）           | 消息流转引擎               |
| **管理层**     | `ConversationManager`（管理会话列表、未读数、当前会话）      | 会话状态管理               |
| **UI 对接**    | 将 `AggregateChatForm` 与 `ConversationManager` 接通（信号槽） | 消息实时显示在界面         |
| **模拟适配器** | `SimPlatformAdapter`——一个内置的模拟器，定时产生假消息或提供一个小对话框让你手动输入"模拟买家消息" | 不依赖千牛即可验证完整链路 |

**Phase 1 完成时的效果**：点一个按钮"模拟新消息" → 聚合界面的会话列表出现新会话 → 点开看到消息气泡 → 输入回复点发送 → 气泡列表出现回复。**整个管线跑通**，不依赖任何外部平台。

### Phase 2：RPA 基础设施（~1 周）

| 任务              | 具体内容                                                     |
| ----------------- | ------------------------------------------------------------ |
| **ScreenCapture** | `PrintWindow` 截图封装，支持按区域裁剪，fallback 到 `BitBlt` |
| **IOCREngine**    | OCR 引擎抽象接口 + `WindowsOCREngine` 实现（WinRT API）      |
| **集成测试**      | 在项目中对千牛窗口截图 → OCR → 输出文字到日志，确认集成正常  |

### Phase 3：千牛消息采集（~1 周）

| 任务                       | 具体内容                                                     |
| -------------------------- | ------------------------------------------------------------ |
| **MessageParser**          | OCR 文字 → 结构化 `ParsedMessage`（区分买家/商家/系统消息）  |
| **IncrementalDetector**    | 文本指纹去重 + 图像哈希快速对比                              |
| **NewMessageDetector**     | 像素条带变化检测 + 窗口标题监控                              |
| **QianniuRPAAdapter (读)** | 实现 `startListening()`，定时 OCR → 检测新消息 → emit `incomingMessage` |

**Phase 3 完成时的效果**：千牛有人发消息 → 聚合界面自动出现该消息。**读方向打通**。

### Phase 4：回复发送（~1 周）

| 任务                       | 具体内容                                                     |
| -------------------------- | ------------------------------------------------------------ |
| **InputSimulator**         | 鼠标点击、键盘按键、组合键、剪贴板操作封装                   |
| **QianniuRPAAdapter (写)** | 实现 `sendMessage()`——切换会话 → 点击输入框 → 粘贴 → 发送 → 验证 |
| **会话切换**               | OCR 识别左侧列表 → 坐标点击切换                              |

**Phase 4 完成时的效果**：在聚合界面输入回复点发送 → 千牛输入框出现文字并发出。**写方向打通。完整闭环。**

### Phase 5：健壮性与体验（~1 周）

弹窗检测、异常恢复、健康指示灯、区域校准工具、自适应轮询、诊断日志等。

---

## 关键设计决策

**暂不引入独立服务端**。当前场景是单机桌面应用 + RPA，所有逻辑（消息路由、数据库、RPA）都在 Qt 桌面进程内完成，用 SQLite 本地存储。这样开发最快、部署最简单。等以后接入 API 或需要多客服协同时，再把 `MessageRouter` 和数据层抽到独立服务端。

### 关键功能

1. 日志系统 — 每条日志同时写入 stderr 和日志文件 `AppData/Demo/AI客服/logs/yyyy-MM-dd.log`，格式：`[时间] [级别] 内容 (文件:行号)`
2. 消息管线 — 从适配器收消息 → 去重 → 自动建会话 → 入库 → 通知 UI
3. 聚合界面 — 左侧会话列表（含未读红点、时间、最后消息预览）、中间消息气泡（收发分左右、可选文字）、右侧客户信息面板
4. 模拟测试 — 点击左上角「模拟消息」按钮即可产生一条随机买家消息，立即出现在会话列表和聊天区
