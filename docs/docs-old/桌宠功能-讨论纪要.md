# 桌宠功能 — 讨论纪要

本文档整理自产品/实现层面关于「桌面宠物」的两轮讨论，供后续设计与开发对齐使用。

---

## 一、背景与目标

- 虽有**帮助中心**，但多数用户未必有耐心通读长文。
- 希望软件更**有趣、有温度**，增加一个桌面宠物（二次元形象），用 Qt 实现较为合适。
- 预期效果：宠物在桌面活动、可互动；用户可通过对话询问「软件有哪些功能、怎么使用」并获得对应回答。

---

## 二、技术可行性（Qt）

### 2.1 整体判断

**可行。** 典型技术组合：

- `Qt::FramelessWindowHint`，必要时配合 `Qt::Tool`、`Qt::WindowStaysOnTopHint`（需权衡是否长期置顶挡界面）。
- `setAttribute(Qt::WA_TranslucentBackground)` + 带 **Alpha 的 PNG**（透明底素材）。
- 使用 `QWidget` / `QLabel` 承载 `QPixmap`，`QTimer` 驱动：
  - **随机移动**：定时修改 `move()`；边界可用 `QScreen::availableGeometry()` 限制在可用工作区内。
  - **眨眼 / 晃耳朵**：多帧图切换；或先做整体轻微缩放/旋转等低成本动画；更精致再考虑多图序列或 Live2D（成本显著上升）。

### 2.2 交互

- **拖动**：`mousePressEvent` / `mouseMoveEvent` 根据全局坐标差更新位置；用移动距离阈值区分「点击」与「拖动」，避免与点击互动冲突。
- **说话气泡**：在宠物头顶或侧方叠一层圆角、半透明 `QWidget`/`QLabel`，定时自动消失；可选用 `QGraphicsOpacityEffect` 做淡入淡出。

### 2.3 资源与动画预期

若当前仅为**单张静态图**，眨眼、挥手等可：

- 补充**切片帧**（眼睛开闭、耳朵角度等）用 `QTimer` 切换；或  
- 先做**整体位移动画**作为过渡，仍能增强「有生命感」。

素材已逐步放入 `resources/pet/`（可继续补全）。

---

## 三、与帮助中心、问答的结合

桌宠对话可作为**帮助内容的分发渠道**，降低「必须读完帮助」的门槛：

| 方式 | 说明 |
|------|------|
| 规则/模板 | 关键词或简单意图 → 固定短答 + 可选「打开帮助中心并跳到某章节」（若 `HelpCenterDialog` 支持按 anchor/章节打开更佳）。 |
| 结构化 FAQ | 资源内嵌 JSON 等维护 Q&A，回答中可写「详情见帮助中心第 × 节」。 |
| 后续扩展 | 再接大模型时，将产品功能摘要写入 system prompt，并约束回答范围；首版不必依赖模型。 |

**结论**：首版用 **FAQ + 跳转帮助中心对应章节** 即可同时做到「有温度」与「有用」，不必一上来接 LLM。

---

## 四、体验与风险（客服场景）

- **干扰**：置顶可能遮挡操作区 → 建议提供**可关闭、托盘、透明度、可选不置顶**等选项；默认行为不要过于激进。
- **多显示器**：游走边界应按宠物所在屏幕的 `availableGeometry()` 计算。
- **性能**：单窗口 + 少量定时器通常很轻；避免不必要的全屏重绘。

---

## 五、与主程序的集成方式

- **同进程**：登录进入主界面后创建桌宠窗口，逻辑简单，与现有工程一致。
- **独立进程**：隔离崩溃、可单独分发——一般**第一期不必**，后续若有需要再拆。

另：**主窗口最小化/隐藏**时，若桌宠锚定在主窗口内会一并不可见；若希望最小化后宠物仍在桌面，需改为**独立顶层窗口**并同步几何——可作为后续迭代。  
**MVP 建议**：先假设**仅主窗口在前台时显示桌宠**，降低复杂度。

## 六、出场与离场方案（已定主方向）

### 6.1 入口

- 在**状态栏**增加 **「桌宠」** 按钮。
- 点击后，在主窗口上出现 **「空间裂缝」** 效果；宠物从裂缝**钻出**，再**跳到主窗口左下角**，并**挥手**（打招呼）。

### 6.2 离场

- 宠物**挥手**，再**淡淡消失**（淡出），与出场形成对称仪式感。

### 6.3 设计上的优点

- 有**叙事感**（召唤而来），区别于普通弹窗，利于记忆入口。
- 落在**主窗口左下角**相对不挡中部业务区；裂缝可布置在偏左/偏下，与落地位置叙事一致。

### 6.4 实现时可预留的问题

1. **裂缝层级**：叠在内容区（如 `QStackedWidget` 之上）较省事；若要做全客户区遮罩 + 洞口，可用半透明层 + 中间镂空/羽化 PNG。
2. **钻出与跳跃**：`QPropertyAnimation` 调整 `pos`/`geometry`，配合 `QEasingCurve::OutBounce` 等即可；素材不全时可用单图 + 位移/缩放过渡。
3. **挥手**：有专用帧最佳；否则可用整体轻微摆动暂代。
4. **按钮行为**：建议 **Toggle**——已在场则触发离场；或区分图标状态表示「召唤 / 送走」，避免重复点击困惑。
5. **可选后期方案**（非首版）：从任务栏/屏幕边缘入场、右键菜单（今日隐藏、降低频率等）。

## 七、MVP 范围建议（供排期）

可优先落地：

- 状态栏「桌宠」入口；裂缝 + 入场/离场动画（可先简化）；左下角锚位。
- 可关闭；宠物与主窗口同生命周期或明确规则。
- 气泡 + 少量固定问答；一键打开帮助中心（及后续按章节锚点跳转）。

后续再扩展：独立顶层窗口、多屏、透明度/置顶策略、更多帧动画、LLM 等。
