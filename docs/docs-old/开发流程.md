## 阶段一：需求调研
**目标**：弄清真实需求，减少假设和返工。

| 步骤 | 任务 | 产出 | 负责人 |
| --- | --- | --- | --- |
| 1.1 | 发放《AI客服需求调研问卷》给老板、电商同事 | 回收问卷 | 开发者 |
| 1.2 | 与主管、客服主管做补充会议，对接需求，确认功能优先级 | 会议记录 | 开发者 |
| 1.3 | 整理问卷与会议结果，提炼关键信息 | 需求总结文档 | 开发者 |
| 1.4 | 识别待确认问题，与相关人员沟通澄清 | 需求确认清单 | 开发者 |


**关键产出**：

+ 问卷回收与统计
+ 需求总结（用户画像、核心操作、痛点、功能优先级）

**完成标准**：明确 MVP 功能、核心用户故事、不做事项。

<font style="color:rgb(25, 27, 31);">MVP构建流程</font>

+ **<font style="color:rgb(25, 27, 31);">第一步：明确产品目标。</font>**<font style="color:rgb(25, 27, 31);">确定AI客服软件到底解决用户的哪些痛点？可以给用户带来什么样的价值？在此之前用户是如何满足自己的需求？</font>
+ **<font style="color:rgb(25, 27, 31);">第二步：定义产品的用户行为路径。</font>**<font style="color:rgb(25, 27, 31);">梳理用户的行为流程，分解用户对产品的具体操作步骤。</font>
+ **<font style="color:rgb(25, 27, 31);">第三步：根据用户路径罗列产品功能。</font>**<font style="color:rgb(25, 27, 31);">帮助用户实现目标需要具备的功能。</font>
+ **<font style="color:rgb(25, 27, 31);">第四步：对上一步列出的功能进行优先级排序。</font>**<font style="color:rgb(25, 27, 31);">列出功能，对功能进行优先级排序，对功能列表做减法和收敛。</font>
+ **<font style="color:rgb(25, 27, 31);">第五步：构建最终功能集合。</font>**<font style="color:rgb(25, 27, 31);">明确好最终功能集合后，可以画出产品原型图，找一些目标用户深入沟通，确定原型终稿，进入开发环节。</font>

## 阶段二：规划与设计
**目标**：形成文档化的产品定义和开发计划，再动手写代码。

| 步骤 | 任务 | 产出 | 负责人 |
| --- | --- | --- | --- |
| 2.1 | 完善 **PRD.md**（产品需求文档） | PRD 定稿 | 开发者 |
| 2.2 | 编写 **TECH_STACK.md**（技术选型） | 技术栈文档 | 开发者 |
| 2.3 | 编写 **APP_FLOW.md**（页面与操作流程） | 流程文档 | 开发者 |
| 2.4 | 设计数据库表结构，写入 **BACKEND_STRUCTURE.md** | 数据结构文档 | 开发者 |
| 2.5 | 设计 UI 规范，写入 **FRONTEND_GUIDELINES.md** | 设计规范 | 开发者 |
| 2.6 | 编写并细化 **IMPLEMENTATION_PLAN.md**（实施计划） | 实施计划定稿 | 开发者 |


**关键产出**：

+ PRD.md 定稿
+ TECH_STACK.md
+ APP_FLOW.md
+ BACKEND_STRUCTURE.md
+ FRONTEND_GUIDELINES.md
+ IMPLEMENTATION_PLAN.md 定稿

**完成标准**：所有文档就绪，开发顺序清晰，无重大遗漏。

---

## 阶段三：开发实现
**目标**：按实施计划分步开发，每步可验证，持续更新进度。

| 步骤 | 任务 | 产出 | 参考文档 |
| --- | --- | --- | --- |
| 3.1 | **阶段 0**：项目初始化、环境搭建、Git 初始化 | 可编译的空项目 | IMPLEMENTATION_PLAN 阶段 0 |
| 3.2 | **阶段 1**：界面骨架（主窗口、布局、占位 UI） | 可运行的界面骨架 | IMPLEMENTATION_PLAN 阶段 1 |
| 3.3 | **阶段 2**：数据层（数据库、DAO、配置存储） | 数据访问层 | IMPLEMENTATION_PLAN 阶段 2 |
| 3.4 | **阶段 3**：核心功能（会话、消息、AI 对接、发送） | MVP 核心功能 | IMPLEMENTATION_PLAN 阶段 3 |
| 3.5 | **阶段 4**：平台对接（至少 1 个电商平台） | 真实收发消息 | IMPLEMENTATION_PLAN 阶段 4 |
| 3.6 | **阶段 5**：扩展与优化（知识库、历史、体验、打包） | 完整可交付版本 | IMPLEMENTATION_PLAN 阶段 5 |


**开发原则**：

+ 小步迭代：完成一步验证一步，再进入下一步
+ 及时更新：每完成一个功能，更新 `progress.txt`
+ 代码规范：遵循 TECH_STACK、FRONTEND_GUIDELINES 等文档
+ 版本控制：每完成一个有效功能后提交 Git

**完成标准**：MVP 核心流程闭环，可在目标环境中安装运行。

---

## 阶段四：交付与迭代
**目标**：交付试用版，收集反馈，持续优化。

| 步骤 | 任务 | 产出 | 负责人 |
| --- | --- | --- | --- |
| 4.1 | 内部试用（老板、电商同事） | 试用反馈 | 使用者 |
| 4.2 | 收集问题与建议 | 问题清单、改进建议 | 开发者 |
| 4.3 | 编写用户手册/操作说明 | 用户文档 | 开发者 |
| 4.4 | 按优先级修复问题、迭代功能 | 新版本 | 开发者 |
| 4.5 | 发布正式版或下一迭代版本 | 安装包/发布物 | 开发者 |


**完成标准**：核心用户能独立使用，反馈问题有记录和排期。
