补充：

“聚合接待”为主要处理客户消息的地方，是与客户的对话处，AI客服系统的核心功能之一。先实现没有AI自动回复功能时，在这里收到客户消息后人工输入消息并回复客户，基础通信流程顺利后，再接入AI实现自动回复。

AI自动回复大致流程：预先设置好对应店铺的回复话术与策略，获取客户询单消息，进行消息处理（相关词汇替换方便AI理解），当AI发出消息时，检查违禁词并替换或删除，最后发送给用户。

## 总结
### 轻简聚合客服介绍
“聚合”是指将所有可支持的电商平台聚合嵌入到软件中，由“聚合接待”统一管理所有平台的客户对话，并支持机器人处理、人工接待等模式切换。在机器人处理模式，AI自动回复客户消息，该功能则是基于机器人管理和知识库系统两大核心功能，本质是作为辅助，提高回复效率。

AI客服的运行原理：

从平台接收到的消息->轻简客服软件（读取平台消息）->AI知识库（通过在知识库搜索客户问题相关的问题，通过引用知识模板，内嵌知识的方式告知AI大模型，大模型通过模拟人工思考的方式计算返回结果）=>轻简客服（发给平台）。

其中，轻简客服扮演的AI知识库的眼睛，与手和脚。负责接收问题，与操作打字输入给客户。而AI大模型，作为一个脑子的角色。用来思考处理问题。

### 核心功能
机器人管理：

创建和管理多个AI客服机器人，每个机器人可以独立配置。

+ 支持创建多个机器人，分别对应不同店铺、平台
+ 提供行业预设模板用于快速创建机器人
+ 支持AI智能分类和关键词匹配

知识库系统：

管理业务知识，主要为AI提供准确的信息检索能力。在机器人配置中可以指定使用哪些知识库分类，实现知识的灵活调用，相当于机器人的“大脑”。常见场景如根据一些客户常问的问题，利用知识库检索，生成回复，再自动回复客户。

AI知识库用来存储自己的内部知识，因为你的产品信息，不属于通用常识，也不属于公共知识。比如你的价格，跟其他人的价格，不一样，如果不告诉AI，AI自带的知识是无法获知的。而正如上面所说，AI是具备公共的知识，以及具备基本的人类的常识。AI知识库的作用，就是将你的知识告诉AI，引导AI根据你的知识+基本常识 来回答。

+ 支持创建多级知识目录
+ 支持商品信息、常见问题、售后政策等多种知识类型
+ 基于向量相似度的语义检索，精准匹配用户问题
+ 支持批量导入导出知识点，提高管理效率

聚合对话接待：

统一管理所有平台的客户对话，支持人机协同处理。

+ 仅转人工、全部接待、人工休息，灵活切换接待策略
+ WebSocket实时推送新消息，不错过任何客户咨询
+ 自动生成AI回复建议，提高客服响应效率（可以直接使用AI建议回复，也可以自己编写。支持发送文本、图片等多种消息类型）
+ 显示历史消息、商品信息、订单状态等完整上下文

其他如消息处理、行话转换、违禁词管理、对话历史、数据备份、日志管理、AI配置等均为辅助功能。

### 预计实现
参照上述，AI客服软件可优先实现如下：

有一个简单的登录/注册界面，进入主窗口之后，从左侧平台列表选择要接待的电商平台，外部的客服窗口（千牛、拼多多等）自动嵌入到本软件中，如果有客户咨询，将消息实时推送到“聚合对话接待”处，查看需要处理的对话，自己编写回复文本，再发送消息给客户。

该流程实现跑通后，中间试用阶段以人工回复为主。

后续再加入机器人管理和知识库管理功能。

如果上面的基础通信流程都未实现，那实现机器人管理和知识库管理功能也只是没有载体的工具，机器人管理和知识库管理只是辅助，用于自动生成AI回复建议，提高回复效率而已，就算没有也可以自己写回复，但相比以前的客服工作方式还是会有不同，以前处理客服咨询，如果有多个电商平台，那要来回切换平台并查看待处理的会话来回复客户，而AI客服软件会将你所有电商平台的客户咨询统一在“聚合对话接待”处管理，也就是所有需要处理的对话被集中在了一起，省去了来回切换电商平台的无用功，该阶段没有AI自动回复，依然是手动回复。

当加入机器人管理和知识库管理功能后，便多了一种机器人处理模式，但仍保留人工接待模式。比如创建一个机器人，对应某个电商平台的店铺，利用知识库系统为AI提供信息检索能力，匹配用户问题，AI自动回复。

上面的核心功能实现后，再选择性实现一些辅助功能。
