# 用户场景与核心流程

## 1. 目标用户

本系统的直接使用者是客服坐席、组长和少量运营支持人员。MVP 阶段的核心用户是客服坐席，其他角色先按辅助角色处理。

### 1.1 客服坐席

主要任务：

- 处理新消息
- 切换会话
- 查看上下文
- 参考 AI 建议回复
- 人工确认发送
- 处理异常降级

### 1.2 组长

主要任务：

- 查看账号状态
- 查看客服处理情况
- 查看异常日志
- 了解平台接入健康状态

### 1.3 运营支持

主要任务：

- 配置快捷回复
- 协助处理异常平台
- 查看基础审计和日志

## 2. 典型工作场景

### 2.1 场景一：正常接待

这是最常见的主流程。

1. 客服打开桌面客户端
2. 客户端加载本地会话列表
3. 平台或模拟源有新消息进入
4. 会话列表出现未读提示
5. 客服点击某个会话
6. 消息区显示最近消息和上下文
7. 客服查看 AI 建议
8. 客服选择建议并编辑
9. 客服手动发送
10. 系统记录发送确认和状态变化

### 2.2 场景二：新消息提醒

当平台产生新消息时，系统需要把消息及时反映给客服。

1. 适配器从模拟源、浏览器页面或桌面可见内容中感知新消息
2. 系统生成统一 `ConversationEvent`
3. 会话列表置顶或未读数变化
4. 客服可点击进入会话
5. 系统记录消息来源和可信度

### 2.3 场景三：AI 辅助回复

AI 的角色是辅助，不是自动接管。

1. 客服打开会话或停留在输入区
2. 客户端收集当前会话最近消息
3. 将上下文发给 Python 服务
4. Python 返回 1-3 条回复建议
5. 客服选择一条建议
6. 文本被填入输入框或复制到剪贴板
7. 客服人工确认后发送

### 2.4 场景四：平台异常降级

当平台采集失败、登录失效或状态不可靠时，系统必须降级而不是继续冒险操作。

1. 适配器检测到窗口失效、DOM 失效或识别失败
2. 平台状态切换为 `degraded` 或 `offline`
3. 自动发送和自动填充关闭
4. 客服仍可查看本地缓存和草稿
5. 系统提示回到原平台人工处理

### 2.5 场景五：模拟平台验证

在接真实平台前，必须先用模拟数据跑通完整链路。

1. 客户端加载模拟会话和消息
2. 产生未读消息和状态变化
3. 客服选择会话并查看消息
4. 请求 AI 建议
5. 发送模拟消息
6. 验证本地缓存、状态机和 UI 刷新

## 3. 核心流程

### 3.1 客服启动流程

1. 启动桌面客户端
2. 检查本地存储可用性
3. 加载最近会话、草稿和账号状态
4. 初始化 Python 服务连接
5. 初始化平台适配器
6. 显示主窗口

### 3.2 收到新消息流程

1. 平台适配器采集到新消息信号
2. 适配器转换为统一事件
3. 核心层做去重和基础校验
4. 消息写入本地缓存
5. 会话未读数更新
6. UI 刷新会话列表和提醒

### 3.3 打开会话流程

1. 客服点击会话列表项
2. 客户端读取本地消息缓存
3. 如有需要，触发历史补充读取
4. 消息区按时间顺序展示
5. 显示会话状态、平台状态、可信度来源

### 3.4 生成 AI 建议流程

1. 客服点击“AI 建议”或进入输入区
2. 客户端收集当前会话上下文
3. 过滤敏感字段和无关信息
4. 调用 Python 服务
5. Python 返回候选回复
6. 客户端展示建议并允许编辑

### 3.5 人工确认发送流程

1. 客服选中一条 AI 建议或手工输入文本
2. 文本进入输入框或剪贴板
3. 客服在目标平台或统一工作台中确认内容
4. 客服点击发送
5. 系统记录发送命令和结果
6. 若发送失败，状态回写为 `failed` 或 `unknown`

### 3.6 异常降级流程

1. 识别到窗口失效、页面变更、OCR 失败或登录失效
2. 适配器触发健康检查失败
3. 平台状态降级
4. 禁用自动填充和自动发送
5. 保留本地消息、草稿和日志
6. 提示客服切回原平台人工处理

## 4. 页面与交互场景

### 4.1 主窗口区域

MVP 主窗口建议包含：

- 左侧会话列表
- 中间消息区
- 右侧 AI 建议或上下文区
- 顶部平台状态和账号状态
- 底部输入区和发送按钮

### 4.2 会话列表交互

会话列表至少支持：

- 选中会话
- 显示未读
- 显示最后消息时间
- 显示平台来源
- 显示状态标记

### 4.3 消息区交互

消息区至少支持：

- 文本消息展示
- 发送方区分
- 时间戳显示
- 状态图标或文本
- 来源可信度提示

### 4.4 AI 建议交互

AI 建议区至少支持：

- 展示 1-3 条建议
- 一键填入输入框
- 一键复制
- 支持人工编辑
- 支持拒绝建议

## 5. 异常和边界场景

### 5.1 登录失效

表现：

- 平台状态异常
- 新消息无法可靠采集

处理：

- 停止自动动作
- 保留草稿和本地缓存
- 提示重新登录原平台

### 5.2 消息不确定

表现：

- OCR 识别不完整
- DOM 状态变化不稳定
- 窗口焦点不明确

处理：

- 记录低可信度消息
- 标记 `unknown`
- 不作为权威历史

### 5.3 发送失败

表现：

- 文本填入后无法确认发送
- 平台返回失败

处理：

- 标记状态为 `failed`
- 保留输入内容
- 不重复自动发送

### 5.4 平台窗口变化

表现：

- 页面结构变更
- 窗口标题变化
- 控件树不可用

处理：

- 适配器健康检查失败
- 自动化能力熔断
- 回退到人工处理

## 6. 核心交互原则

- AI 只建议，不自动发送
- 用户始终保留最终确认权
- 失败优先降级，不盲目重试
- 任何平台细节都不直接进入 UI 业务层
- 所有不确定状态都要显式展示
- 文案必须清楚，避免客服误判
