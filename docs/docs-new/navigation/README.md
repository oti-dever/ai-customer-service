# Windows 桌面客服工作台系统文档说明

## 1. 文档目的

`docs/docs-new` 用于沉淀 Windows 桌面客服工作台系统的当前主线设计、MVP 范围、工程边界和执行进度。早期开发前规划已经基本完成，后续维护重点是让文档与代码实现、M5 PoC 结果和验收清单保持一致。

当前项目重点是先完成 MVP：

- Windows 桌面客服工作台
- C++ / Qt 客户端
- Python AI 辅助服务
- 统一会话与消息模型
- 单平台真实 PoC
- 非官方 API / SDK 条件下的可见内容采集
- AI 只建议，不自动发送

## 5. 当前开发建议

当前文档已经足够继续推进 MVP。按最新进度，M0-M4 主链路已经基本完成：

- Qt 工程骨架、统一模型、模拟平台主流程已建立
- 状态机、本地缓存、草稿和最近会话恢复已接入
- C++ IPC 与 Python AI sidecar 已打通
- UI 已迁移到统一模型信号入口
- Python RPA 目录已开始向 `core/`、`platforms/`、`db/` 收敛

下一步建议聚焦 M5 单平台真实 PoC：

1. 选定一个真实平台，优先拼多多 Web，备选千牛 PC
2. 验证平台可用状态、可见会话和可见消息采集
3. 将采集结果转换为统一 `Conversation` / `Message`
4. 在 C++ 聚合工作台展示真实平台消息
5. 复用 Python AI 建议链路生成回复草稿
6. 默认只做草稿回填或复制到剪贴板，最终发送由人工确认
7. 记录降级状态、`unknown` 状态和验收结果

## 6. 暂不建议推进的方向

MVP 阶段暂不建议推进：

- 多平台同时接入
- 个人微信 Hook / DLL 注入
- 未授权抓包 / 中间人代理
- 无人值守自动发送
- 完整插件热更新
- 完整多租户 SaaS 架构
- 完整 BI / 数据仓库
