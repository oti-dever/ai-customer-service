# 客户端缓存库与服务事实库

## 目标

客户端和 Python 服务不再长期共写同一个 SQLite 文件。

- C++ 客户端使用本地缓存库：`client_cache.db`
- Python 服务使用事实库：`service.db`
- 两端通过 HTTP / WebSocket 同步数据，不通过共享 SQLite 文件同步

## 默认路径

### C++ 客户端缓存库

默认位于 Qt AppData 目录：

- Windows: `%APPDATA%/YangYangAI/CustomerServiceDemo/client_cache.db`
- macOS: `~/Library/Application Support/YangYangAI/CustomerServiceDemo/client_cache.db`
- Linux: `$XDG_DATA_HOME/YangYangAI/CustomerServiceDemo/client_cache.db`

客户端缓存库保存 UI 快速恢复所需的数据，例如：

- 会话列表缓存
- 当前会话消息缓存
- 草稿
- 本地 UI 状态
- pending / sent / failed 发送状态的本地展示副本

客户端缓存库是可重建缓存，不是平台事实源。

### Python 服务事实库

默认位于仓库：

```text
database/service.db
```

可通过环境变量覆盖：

- `AI_CUSTOMER_SERVICE_SERVER_DB`
- `AI_CUSTOMER_SERVICE_SNAPSHOT_DB`
- `AI_CUSTOMER_SERVICE_DB`

服务事实库保存平台观察和发送链路的权威数据，例如：

- `rpa_events` 持久事件日志
- 平台会话
- 平台消息
- 出站发送命令和结果
- replay / snapshot 所需状态

## 同步流程

实时消息：

```text
平台窗口/RPA -> Python observer -> service.db -> WebSocket event -> C++ client_cache.db -> UI
```

启动或断线恢复：

```text
C++ 先读 client_cache.db 快速展示
-> 调 /api/platform/replay 补事件
-> 调 /api/cache/snapshot 补快照
-> upsert 到 client_cache.db
-> 刷新 UI
```

发送消息：

```text
C++ client_cache.db 写 pending
-> Python send_message
-> Python service.db 写 outbound command / result
-> RPA 执行发送
-> Python 推 message_sent / send_failed
-> C++ client_cache.db 更新 sent / failed
```

## 迁移

旧版统一库默认是：

```text
database/app.db
```

一次性拆分可运行：

```powershell
cd python
python -m service.migrate_legacy_databases --target both
```

默认行为：

- 从 `database/app.db` 复制出 `database/service.db`
- 同时复制出 AppData 下的 `client_cache.db`
- 已存在目标库时不覆盖

如需覆盖：

```powershell
cd python
python -m service.migrate_legacy_databases --target both --overwrite
```

C++ 客户端首次启动时，如果 AppData 下还没有 `client_cache.db`，会优先从旧 AppData `app.db` 复制迁移；如果不存在，再尝试从仓库 `database/app.db` 复制。

## 约束

- Python 服务不再探测或默认打开 Qt AppData 下的客户端库。
- C++ 客户端不直接打开 Python 服务事实库。
- 跨进程同步只走 HTTP / WebSocket。
- `client_cache.db` 可以删除后由 replay / snapshot 重建。
- `service.db` 是平台事实库，需要作为运维备份对象。
