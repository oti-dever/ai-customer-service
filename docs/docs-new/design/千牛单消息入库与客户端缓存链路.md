# 千牛单消息入库与客户端缓存链路

本文结合当前代码实现，梳理千牛平台收到一条新的未读消息时，Python 服务端事实库和 C++ 客户端缓存库的写入顺序、关键字段来源、表之间的关联。

场景假设：

- 千牛未读会话显示名：`tb4947894539`
- 对方发给我一条文本消息：`你好`
- 未读会话里只读取到这一条消息
- 服务端事实库中原来没有该会话，也没有该消息
- 客户端缓存库中原来也没有该会话和该消息
- 默认千牛账号：`local_qianniu`

## 一、事件生成

入口是 `python/rpa/platforms/qianniu/adapter.py` 的 `_scan_unread_and_fetch()`。

当前代码顺序是：

1. `read_visible_sessions(limit=100, detect_unread=True)` 读取可见会话列表，并筛出 `item.unread == true` 的会话。
2. 选择第一个未读会话 `target = tb4947894539`。
3. `select_session(target)` 切换到该会话。
4. 先生成并 append 一个 `conversation_observed` 事件。
5. 再读取当前聊天区消息。
6. 把读取到的消息逐条转换成 `message_observed` 事件。
7. 经 `_filter_unread_message_events()` 去重/过滤后逐条 append。

因此，本场景通常会产生两个 RPA 事件：

```text
conversation_observed
message_observed
```

关键字段生成规则：

- `conversation_key = qianniu:<account_id>:<display_name>`，本例为 `qianniu:local_qianniu:tb4947894539`。
- `event_id = evt_ + sha1(event_type|conversation_key|occurred_at|payload)[:24]`。
- `seq/cursor` 不是 adapter 生成的，而是 `RpaEventStore.append()` 写入服务端事件队列前生成。
- `message_observed.payload.platform_msg_id = qianniu_ + sha1(raw_id)[:24]`。
- 千牛消息 `raw_id` 由 `qianniu|account_id|display_name|timestamp|sender|direction|text` 拼出。

本例消息事件的核心 payload 形态：

```json
{
  "platform_msg_id": "qianniu_<sha1前24位>",
  "display_name": "tb4947894539",
  "direction": "inbound",
  "sender_role": "customer",
  "sender_name": "tb4947894539",
  "content_type": "text",
  "content": "你好",
  "source_type": "ui_observed",
  "confidence": 70,
  "verification_status": "unverified",
  "metadata": {
    "observation_method": "uia",
    "timestamp": "<千牛界面时间文本>",
    "status": "<解析出的状态>",
    "raw": "<原始解析文本>"
  }
}
```

注意：payload 里的 `direction` 是千牛解析侧原始方向，常见为 `inbound/outbound`；写入统一 `messages.direction` 时会被归一成 `in/out/system`。

## 二、服务端事实库写入

服务端入口是 `python/service/rpa_bridge.py` 的 `RpaEventStore.append()`。它会先给事件写入：

- `seq`：服务端内存事件队列自增序号。
- `cursor`：当前实现等于 `str(seq)`。

随后调用 `PythonServiceTruthStore.persist_event()`。只有持久化没有被过滤时，事件才会进入内存队列并广播给客户端。`rpa_events.raw_event_json` 中已经包含 `seq/cursor`，但不包含后续才加上的 `truth_persisted=true`。

### 1. conversation_observed

`persist_event()` 对 `conversation_observed` 的顺序：

1. `_ensure_schema()`
2. `_should_accept_observed_event()`
3. `_append_event_log()`
4. `_upsert_conversation()`
5. `_upsert_platform_conversation()`
6. `commit()`
7. `RpaEventStore.append()` 广播事件

#### rpa_events

先插入事件日志：

- `event_id = evt_<...>`
- `event_type = conversation_observed`
- `platform = qianniu`
- `account_id = local_qianniu`
- `conversation_key = qianniu:local_qianniu:tb4947894539`
- `occurred_at = conversation_event.occurred_at`
- `payload_json = conversation_observed.payload`
- `raw_event_json = 完整事件 JSON`
- `source_role = python_service_truth`

`event_id` 有唯一约束，语句是 `INSERT OR IGNORE`，重复事件不会重复插入日志。

#### conversations

因为服务端原来没有该会话，插入统一会话记录：

- `id = server_conv_id`，服务端事实库自增主键
- `platform = qianniu`
- `platform_conversation_id = qianniu:local_qianniu:tb4947894539`
- `account_id = local_qianniu`
- `customer_name = tb4947894539`
- `last_message = ''`，会话观察事件没有 `content`
- `last_time = conversation_event.occurred_at`
- `unread_count = 0`
- `status = active`
- `updated_at = conversation_event.occurred_at`

唯一键是 `(platform, platform_conversation_id)`。

#### qianniu_conversations

随后插入千牛会话扩展记录：

- `conversation_id = server_conv_id`
- `qianniu_account_id = local_qianniu`
- `qianniu_conversation_key = qianniu:local_qianniu:tb4947894539`
- `display_name = tb4947894539`
- `last_unread_badge = 0`，当前 conversation payload 没有顶层 `unread_count`
- `last_observed_at = conversation_event.occurred_at`
- `last_health_status = ''`
- `raw_payload_json = conversation_observed.payload`

`qianniu_conversations.conversation_id` 是唯一键，并外键关联 `conversations.id`。

### 2. message_observed

`persist_event()` 对 `message_observed` 的顺序：

1. `_ensure_schema()`
2. `_should_accept_observed_event()`
3. `_append_event_log()`
4. `_upsert_conversation()`
5. `_upsert_platform_conversation()`
6. `_upsert_message()`
7. `_upsert_platform_message()`
8. `commit()`
9. `RpaEventStore.append()` 广播事件

#### rpa_events

插入第二条事件日志：

- `event_type = message_observed`
- `platform = qianniu`
- `account_id = local_qianniu`
- `conversation_key = qianniu:local_qianniu:tb4947894539`
- `payload_json = message_observed.payload`
- `raw_event_json = 完整消息事件 JSON`

#### conversations

`_upsert_conversation()` 先按：

```sql
platform = 'qianniu'
platform_conversation_id = 'qianniu:local_qianniu:tb4947894539'
```

找到刚才由 `conversation_observed` 插入的 `server_conv_id`，然后更新：

- `account_id = local_qianniu`
- `customer_name = tb4947894539`
- `last_message = 你好`
- `last_time = message_event.occurred_at`
- `status = active`
- `deleted_at = NULL`
- `updated_at = message_event.occurred_at`

服务端这里不会把 `unread_count` 加 1；当前服务端事实库在该链路下仍是 `0`。

#### qianniu_conversations

同一个 `conversation_id = server_conv_id` 再次 upsert：

- `qianniu_account_id = local_qianniu`
- `qianniu_conversation_key = qianniu:local_qianniu:tb4947894539`
- `display_name = tb4947894539`
- `last_unread_badge = 0`
- `last_observed_at = message_event.occurred_at`
- `raw_payload_json = message_observed.payload`

因此本例最终服务端 `qianniu_conversations.raw_payload_json` 会从会话观察 payload 被覆盖成消息 payload。

#### messages

`_upsert_message()` 先按 `platform_message_id` 查重；本例原来不存在，所以插入统一消息记录：

- `id = server_msg_id`，服务端事实库自增主键
- `conversation_id = server_conv_id`
- `platform_message_id = qianniu_<sha1前24位>`
- `client_message_id = ''`
- `direction = in`，由 `inbound/customer` 归一得到
- `sender = customer`
- `sender_name = tb4947894539`
- `content_type = text`
- `content = 你好`
- `status = observed`
- `error_reason = ''`
- `message_time = message_event.occurred_at`
- `created_at = message_event.occurred_at`
- `updated_at = message_event.occurred_at`

`messages.conversation_id` 外键关联 `conversations.id`。

#### qianniu_messages

随后插入千牛消息扩展记录：

- `message_id = server_msg_id`
- `conversation_id = server_conv_id`
- `qianniu_account_id = local_qianniu`
- `qianniu_conversation_key = qianniu:local_qianniu:tb4947894539`
- `qianniu_display_name = tb4947894539`
- `platform_message_id = qianniu_<sha1前24位>`
- `direction = inbound`，保留 payload 原始方向
- `sender_role = customer`
- `raw_sender = tb4947894539`
- `raw_timestamp_text = message_event.occurred_at`，当前服务端实现优先 `payload.original_timestamp`，没有则使用事件时间；不会使用 `metadata.timestamp`
- `parser_source = uia`
- `source_type = ui_observed`
- `confidence = 70`
- `verification_status = unverified`
- `original_timestamp = message_event.occurred_at`
- `content_image_path = ''`
- `evidence_ref = ''`
- `raw_payload_json = message_observed.payload`

`qianniu_messages.message_id` 是唯一键并外键关联 `messages.id`；`conversation_id` 外键关联 `conversations.id`。

## 三、推送给客户端后的缓存库写入

客户端实时事件入口是 `src/services/platforms/qianniurp_adapter.cpp` 的 `QianniuRPAAdapter::handleRpaEvent()`。

重要差异：当前客户端 `QianniuRPAAdapter::normalizeConversationKey()` 会把服务端完整 key：

```text
qianniu:local_qianniu:tb4947894539
```

裁剪成：

```text
tb4947894539
```

所以实时事件路径下，客户端缓存库的 `conversations.platform_conversation_id` 和 `qianniu_conversations.qianniu_conversation_key` 会写短 key `tb4947894539`。服务端事实库写的是完整 canonical key。两个库的自增 ID 也彼此独立，不要求一致。

### 1. conversation_observed

客户端顺序：

1. `QianniuRPAAdapter::handleRpaEvent()`
2. `emitConversationObserved(event)`
3. 发出 `conversationObserved(info)`
4. `MessageRouter::onConversationObserved(info)`
5. `ConversationDao::findByPlatformId(platform, platformConversationId)`
6. `ConversationDao::upsertObservedCacheConversation(unified)`
7. `QianniuConversationDao::upsertConversation(...)`
8. 发出会话创建/更新信号

#### conversations

本例客户端缓存库原来没有该会话，所以插入：

- `id = client_conv_id`，客户端缓存库自增主键
- `platform = qianniu`
- `platform_conversation_id = tb4947894539`
- `account_id = local_qianniu`
- `customer_name = tb4947894539`
- `last_message = ''`
- `last_time = datetime('now','localtime')`，`ConversationDao::create()` 当前没有使用事件时间写入
- `unread_count = 0`
- `status = active`
- `cache_scope = local_cache`
- `cache_origin = platform_observed_cache`
- `updated_at = datetime('now','localtime')`

唯一键是 `(platform, platform_conversation_id)`。

#### qianniu_conversations

随后插入千牛会话扩展：

- `conversation_id = client_conv_id`
- `qianniu_account_id = local_qianniu`
- `qianniu_conversation_key = tb4947894539`
- `display_name = tb4947894539`
- `last_unread_badge = 0`
- `last_observed_at = datetime('now','localtime')`
- `last_health_status = ''`
- `raw_payload_json = {}`，`onConversationObserved()` 调用扩展 DAO 时没有传 payload

### 2. message_observed

客户端顺序：

1. `QianniuRPAAdapter::handleRpaEvent()`
2. `platformMessageFromEvent(event)` 把事件转换为 `PlatformMessage`
3. 发出 `incomingMessage(msg)`
4. `MessageRouter::onIncomingMessage(msg)`
5. `MessageDao::existsByPlatformMsgId(platformMsgId)` 查重
6. `ensureConversation(msg)` 查找或补建会话
7. `createUnifiedMessage(msg, convId)`
8. `MessageDao::createObservedCacheMessage(unifiedMessage)`
9. `QianniuConversationDao::upsertConversation(...)`
10. `ConversationDao::updateLastMessage(...)`
11. 入方向消息时 `ConversationDao::incrementUnread(...)`
12. 发出 `unifiedMessageReceived`
13. `QianniuConversationDao::createMessageExtension(...)`
14. `MessageDao::findById(msgId)`
15. 发出 `messageReceived` 和会话更新信号

#### conversations

`ensureConversation()` 用：

```text
platform = qianniu
platform_conversation_id = tb4947894539
```

找到刚才插入的 `client_conv_id`，不会再插入会话。

消息写入后更新：

- `last_message = 你好`
- `last_time = message_event.occurred_at`，来自 `msg.createdAt`
- `unread_count = unread_count + 1`，本例最终为 `1`
- `updated_at = datetime('now','localtime')`

#### messages

查重通过后，插入客户端统一消息记录：

- `id = client_msg_id`，客户端缓存库自增主键
- `conversation_id = client_conv_id`
- `platform_message_id = qianniu_<sha1前24位>`
- `client_message_id = ''`
- `direction = in`
- `sender = customer`
- `sender_name = tb4947894539`
- `content_type = text`
- `content = 你好`
- `status = observed`
- `error_reason = ''`
- `message_time = message_event.occurred_at`
- `cache_scope = local_cache`
- `cache_origin = platform_observed_cache`
- `created_at = message_event.occurred_at`
- `updated_at = message_event.occurred_at`

`messages.conversation_id` 外键关联客户端 `conversations.id`。

#### qianniu_conversations

`onIncomingMessage()` 会再次 upsert 客户端千牛会话扩展：

- `conversation_id = client_conv_id`
- `qianniu_account_id = local_qianniu`，从 payload 里的 `_event_account_id` 取值
- `qianniu_conversation_key = tb4947894539`，因为调用时显式传入的是客户端短 key
- `display_name = tb4947894539`，优先取 payload.display_name
- `last_unread_badge = 0`
- `last_observed_at = datetime('now','localtime')`
- `raw_payload_json = message_observed.payload + 客户端补充的 _event_account_id/_event_conversation_key/raw_direction/raw_sender_role/normalized_direction`

因此客户端最终 `qianniu_conversations.raw_payload_json` 也会被消息 payload 覆盖；但 `qianniu_conversation_key` 仍是短 key。

#### qianniu_messages

最后插入客户端千牛消息扩展：

- `message_id = client_msg_id`
- `conversation_id = client_conv_id`
- `qianniu_account_id = local_qianniu`
- `qianniu_conversation_key = tb4947894539`
- `qianniu_display_name = tb4947894539`
- `platform_message_id = qianniu_<sha1前24位>`
- `direction = inbound`，保留 payload 原始方向
- `sender_role = customer`
- `raw_sender = tb4947894539`
- `raw_timestamp_text = ''`，客户端 DAO 当前只读 `payload.original_timestamp`，不读 `metadata.timestamp`
- `parser_source = uia`
- `source_type = ui_observed`
- `confidence = 70`
- `verification_status = unverified`
- `original_timestamp = ''`
- `content_image_path = ''`
- `evidence_ref = ''`
- `raw_payload_json = message payload`

`qianniu_messages.message_id` 唯一并外键关联客户端 `messages.id`；`conversation_id` 外键关联客户端 `conversations.id`。

## 四、最终关系汇总

服务端事实库：

```text
rpa_events
  conversation_observed(event_id=evt_...)
  message_observed(event_id=evt_...)

conversations(server_conv_id)
  platform = qianniu
  platform_conversation_id = qianniu:local_qianniu:tb4947894539
  last_message = 你好
  unread_count = 0

qianniu_conversations
  conversation_id -> conversations.id(server_conv_id)
  qianniu_conversation_key = qianniu:local_qianniu:tb4947894539

messages(server_msg_id)
  conversation_id -> conversations.id(server_conv_id)
  platform_message_id = qianniu_<sha1前24位>
  direction = in
  content = 你好

qianniu_messages
  message_id -> messages.id(server_msg_id)
  conversation_id -> conversations.id(server_conv_id)
  direction = inbound
```

客户端缓存库：

```text
conversations(client_conv_id)
  platform = qianniu
  platform_conversation_id = tb4947894539
  last_message = 你好
  unread_count = 1
  cache_origin = platform_observed_cache

qianniu_conversations
  conversation_id -> conversations.id(client_conv_id)
  qianniu_conversation_key = tb4947894539

messages(client_msg_id)
  conversation_id -> conversations.id(client_conv_id)
  platform_message_id = qianniu_<sha1前24位>
  direction = in
  content = 你好
  cache_origin = platform_observed_cache

qianniu_messages
  message_id -> messages.id(client_msg_id)
  conversation_id -> conversations.id(client_conv_id)
  direction = inbound
```

跨库稳定关联字段：

- 会话：服务端使用完整 key `qianniu:local_qianniu:tb4947894539`；客户端实时缓存当前使用短 key `tb4947894539`。这两者不是同一个字符串。
- 消息：服务端和客户端都使用同一个 `platform_message_id = qianniu_<sha1前24位>`。
- 事件：只有服务端事实库有 `rpa_events`；客户端缓存库不保存 `rpa_events`。
- 自增 ID：`server_conv_id/server_msg_id` 与 `client_conv_id/client_msg_id` 分属不同 SQLite 库，不能互相比较。
